## 既知の限界

1. **徒歩は直線距離×1.3の近似**。川や線路で分断された場所は実際より甘く出る
2. **出発点はメッシュ中心**。500m四方の中の位置差(最大350m≒徒歩6分)は丸められる
3. **特定の1日(平日)のダイヤで計算**。冬ダイヤ・減便期は別途再計算が必要
4. デマンド交通・タクシー・鉄道駅の構内徒歩などは対象外
   (山形鉄道フィードは読み込むが、駅とバス停の乗換は300m徒歩ルールで近似)
5. OSMのスーパー網羅性は地域差がある(第1部の調査で山形市内は実用的な水準と確認済み)
6. **JR在来線はオープンデータとして提供されておらず、本マップの到達時間はバスのみで計算している。**
   かみのやま温泉〜山形など鉄道が実質的な移動軸である区間では、実際より不利に評価される
7. **モビリティ・カルテ(`webapp/karte/`)の位置も上記2と同じメッシュ中心の近似**。
   住所やGPSで選ばれるのは「おうちのあたり(約500m四方)」のめやすで、1軒ごとの
   正確な数字ではない
8. カルテの住所検索は国土地理院の外部API(無料・キー不要)に依存する。
   停止・仕様変更のリスクがあり、その場合もGPS・地区選択の2経路で使い続けられる設計にした
9. カルテの総合評価(A〜E)は病院所要時間・通院可否からの機械的なめやすで、
   坂道・冬道・個人の歩行速度などは反映していない

## モビリティ・カルテのプライバシー

`webapp/karte/` は住所やGPSから「あなたの家の移動のようす」を表示する機能。
入力した住所は**この端末(ブラウザ)の中だけで使い、保存も自サーバーへの送信もしない**。
候補地点を調べるときだけ、国土地理院の住所検索API(https://www.gsi.go.jp/johofukyu/johofukyu41022.html)
に住所の文字列を送る。それ以外の通信先はGitHub Pages(静的ファイルの配信)のみで、
サーバー側で個人の入力を記録する仕組み自体を持たない。
